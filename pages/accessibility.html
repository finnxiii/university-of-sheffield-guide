General Accessibility Features

Our website is designed to prioritize accessibility, ensuring that it’s usable for all users, including those with disabilities. Key features implemented to enhance accessibility include semantic HTML5 elements, such as <header>, <nav>, <main>, <section>, <article> and <footer>, which assist screen readers in interpreting the page structure, as well as ARIA (Accessible Rich Internet Applications) roles where necessary to clarify element functions (W3C, 2017).

Text Readability and Color Contrast

For text readability, we chose responsive font sizing (using em unit) and high-contrast color schemes, ensuring content visibility across different devices and for users with low vision or color blindness. According to WebAIM (2020), these practices make it easier for users to customize font size and improve text legibility on various screens.

Keyboard Accessibility and Navigation

Keyboard accessibility is ensured by intuitive tab ordering and focus styles for navigation links, enabling users who cannot use a mouse to navigate smoothly through the site. We also ensured sufficient color contrast (minimum 4.5:1 ratio) for readability, as recommended by WCAG (Web Content Accessibility Guidelines) 2.1.

Screen Reader Support and Resources

Screen readers like NVDA and VoiceOver assist users further by providing audio feedback for visual information. For users unfamiliar with screen readers, we provide links to NVDA and VoiceOver resources.

Additional Accessibility Consideration and Resources

Despite these efforts, screen readers and alternative input methods may handle additional accessibility needs. For advice on compatible tools and services, users may refer to the W3C accessibility resource page.
