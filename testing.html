Performance Optimization

To enhance the website’s performance, I chose JPG and JPEG image formats over PNG. These formats provide good image quality while significantly reducing file sizes, leading to faster loading times. For the Google Maps embedded HTML, I implemented lazy loading, ensuring that the map only loads when it becomes visible on the user’s screen. This approach conserves bandwidth and accelerates the initial page load, especially beneficial for mobile users and those on slower connections.

Code Validation and Debugging

HTML and CSS validators were used to ensure code accuracy and compliance with web standards. The W3C HTML Validator and CSS Validator helped identify syntax errors and compatibility issues.

The HTML validation results showed some minor issues, such as unclosed tags, and a <div> tag being in <label> tag. The latter was resolved by removing <div> tag, arranging css properties to obtain the previous design.

The CSS validation found an issue in one of the CSS files, main.css, as above. It was resolved by adding appropriate unit after the value.

Cross-Browser and Device Compatibility Testing

To ensure consistent appearance and functionality, the website was tested on multiple devices and browsers including Chrome, Safari and Arc. I mainly used Chrome's Developer Tools for testing different screen sizes, from mobile to desktop views, and debugging. I emulated on different devices, such as iPhone 14 Pro Max, Samsung Galaxy S20 Ultra and iPad Air, verifying that layouts adapted smoothly across resolutions.

Accessibility Improvements

Accessibility was a key consideration, and the WAVE Accessibility Tool was used to identify accessibility issues. The tool highlighted an error, which was an empty form label due to the CSS checkbox hack in desktop screen, and a few warning, which was due to the heading usage and very small font size of the copyright text. The label error was ignored due to the intentional use of the hack.

Additionally, the test on colour contrast found no issues, and was verified to meet Web Content Accessibility Guidelines (WCAG) standards.
